# Server：配对与鉴权

## 职责

管理短期配对码、设备注册、设备密钥、设备撤销与受保护 API 的认证。

## 特性与功能点

- `PAIR-01`：Android 发起配对时 Server 生成短期配对码，localhost 管理端负责查看。
- `PAIR-02`：Web/Android 使用地址、配对码和设备名完成配对。
- `PAIR-03`：连续失败触发限流或短期锁定。
- `AUTH-01`：已配对客户端通过 Bearer 设备密钥访问受保护 API。
- `AUTH-02`：设备可以重命名、撤销；撤销后请求和活动状态流失效。
- `AUTH-03`：敏感值不在非 localhost 响应、日志或测试证据中回显。
- `AUTH-04`：localhost 管理页通过独立 SSE 接收配对和设备快照失效通知；事件不携带配对码或设备密钥，断线重连后重新读取快照。

## 关键约束

- 配对码当前为四位十进制数字，默认五分钟有效。
- 非 localhost 客户端不得取得真实配对码。
- 本机设备存储属于运行状态，不是源码资产。
- localhost 管理页空闲时不轮询配对或设备列表；仅在真实状态变化、事件重连或活动配对到期时刷新。
- 设备 `last_seen` 不驱动管理事件，避免远程鉴权活动造成管理页刷新风暴。

## 主要测试

参见 `TC-PAIR-*`、`TC-AUTH-*` 和 `TC-XPLAT-*`。
