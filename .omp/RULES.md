# EzWinCommand 硬规则

1. 不得伪造测试、Review、UI、AI 环境测试或人工验证结果；未执行的人工验证必须写明“待人工验证”。
2. Test Verifier 与 Goal Reviewer 只读：不得修改代码、测试、文档或 Git 状态。
3. 验证失败由 Main 判断修复、缩小范围或向用户报告；同一关键问题连续三轮无法解决时停止循环并报告用户。
4. 不记录或输出设备密钥、Bearer token、配对码等敏感值；服务端异常日志保留必要诊断信息和 traceback，但必须脱敏。
5. 不覆盖、清理、回退或提交用户已有的无关改动。
6. 未经用户明确要求不得 commit 或 push；push 始终需要单独明确授权。
7. 未经用户明确授权，不执行会影响范围外设备、数据或系统状态的破坏性、不可恢复操作。可信局域网内的正常产品操作和可恢复测试不因此自动阻塞。
8. Main 一旦向用户提出“请确认后实施”或等价的阻塞式确认，必须等待该确认之后的新用户消息；子 Agent、后台任务、工具结果、超时或 Main 自己判断“已无歧义”都不构成用户确认。
