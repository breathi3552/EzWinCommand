# 产品与测试资产维护

长期资料只承载无法从代码、搜索和测试目录可靠恢复的事实。候选交付进入独立验证前，按需更新 `docs/product/`、`docs/tests/coverage-map.md`、`docs/versions/` 和 `docs/changelog.md`。历史 `docs/v*/` 默认不读取；不要为每轮任务创建需求目录或复制测试日志。

## 产品契约

每个模块记录：

- 职责与边界。
- 产品边界与安全不变量。
- 用户可观察且必须保持的特殊行为。
- 环境与人工验证边界。

不维护普通功能清单、代码模块地图、类/函数导航或调用方列表。

## 协议契约

保留 Server、Web、Android 的稳定 wire，请求/响应/错误语义，状态机与恢复规则，设备撤销与 SSE 行为，以及兼容和破坏性变更边界。

## 验收覆盖与缺口

`docs/tests/coverage-map.md` 只维护业务主路径或稳定契约、自动化层级、可复核证据和当前缺口。普通单元测试由 Test Verifier 动态枚举，不在文档中维护静态测试文件列表。稳定 ID 只在需求契约、自动化证据和 Verifier 持续引用时保留；没有追溯消费者的编号应改为清晰标题。

## 版本日志

在 `docs/versions/` 或 `docs/changelog.md` 按版本或日期记录增量：新增、变更、修复、移除和待人工验证。不要维护内部过程流水账。
