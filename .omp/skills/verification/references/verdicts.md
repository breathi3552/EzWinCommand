# 验证结论协议

## Test Verifier

```text
结论：通过 | 通过（待人工验证） | 失败 | 阻塞
目标与风险覆盖：充分 | 不充分 | 未知
自动化检查：...
AI 环境检查：...
人工验证：无 | 待人工验证：...
失败项：无 | ...
证据：...
返回对象：目标审查 | Main | 用户
```

结论约束：

- “通过”要求本轮目标和已识别风险覆盖充分，选择的检查已完成，人工验证为“无”。
- “通过（待人工验证）”要求自动化证据足以支持已执行范围，未自动化项目有明确人工覆盖范围，不把未执行项冒充结果。
- 覆盖“不充分”且影响目标时返回“失败”；关键事实未知、无法形成可靠结论时返回“阻塞”。
- 只有未授权的破坏性、不可恢复操作或真实凭据风险阻止继续验证；可信局域网内正常操作、localhost、可恢复测试和普通人工待办不自动阻塞。

## Goal Reviewer

```text
结论：通过 | 失败 | 阻塞
目标覆盖：G-1=已覆盖 | 缺失 | 不明确；...
验收覆盖：AC-1=已覆盖 | 缺失 | 不明确；...
无目标映射的变更：无 | ...
遗漏范围：无 | ...
非目标影响：无 | ...
产品资料漂移：无 | ...
测试结论支撑目标：充分 | 不充分 | 未知；...
证据：...
返回对象：Release | Main | 用户
```

## 汇总

- Test“通过”且 Goal“通过” => 总结论“通过”。
- Test“通过（待人工验证）”且 Goal“通过” => 总结论“通过（待人工验证）”。
- 任一“失败” => Main 修复后重验。
- 任一“阻塞” => 向用户报告阻塞。
- 同一关键失败连续三轮无法解决 => 停止循环并报告用户。
- 缺少任一规定字段或字段值互相矛盾 => 不得放行，返回“阻塞”。
- Goal Reviewer 判断测试结论“不充分”时返回“失败”，判断“未知”时返回“阻塞”。
